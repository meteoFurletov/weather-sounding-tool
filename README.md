# Обработчик Метеорологических Зондирований

Этот инструмент обрабатывает данные метеорологических зондирований из базы данных Университета Вайоминга. Он извлекает температурные инверсии и сохраняет их в файлах Excel для дальнейшего анализа.

> ℹ️ **Изменение сайта Вайоминга (2025 год).**
> Архив переехал на новый сервер `weather.arcc.uwyo.edu`. Прежняя выгрузка списком за целый месяц исчезла — сайт отдаёт данные по одному сроку и работает медленно (~5 секунд на срок).
>
> Программа обновлена и по-прежнему загружает данные **автоматически**: сначала за один запрос она получает список всех доступных сроков за год (инвентарь станции), затем скачивает только существующие сроки параллельно, с повторами при сбоях. Пользоваться нужно так же, как раньше — указать год, месяц и станцию. Загрузка одного месяца занимает около минуты.

## Особенности

- Загрузка данных зондирования из базы данных Университета Вайоминга
- Работа с локальными файлами для предотвращения проблем с соединением
- Обнаружение температурных инверсий на высоте до 1000 метров
- Создание отдельных файлов Excel для каждого зондирования
- Формирование общего файла анализа с несколькими листами
- Несколько вариантов использования (онлайн и локально)

## Автоматическая загрузка с нового сайта

Программа сама скачивает данные с нового сайта. Так как выгрузки за месяц там больше нет, она берёт данные так:

1. один запрос — **инвентарь станции за год** (список всех существующих сроков и их источник `src`);
2. затем скачивает только эти сроки **параллельно**, с повторами при таймаутах, и объединяет их.

Пустые сроки не запрашиваются вовсе, а источник (`src`) для каждой станции берётся из самого инвентаря — угадывать ничего не нужно. Один месяц загружается примерно за минуту (сервер отдаёт ~5 секунд на срок).

### Способы запуска

**Google Colab** (рекомендуется): откройте `demo_ru.ipynb`, выполните раздел «Настройка», задайте год/месяц/станцию и запустите раздел «3. Загрузка и обработка».

**Командная строка:**

```bash
# один месяц
python weather_tool.py --year 2025 --month 1 --station 26075

# диапазон месяцев
python weather_tool.py --year 2025 --month 1 --end-month 3 --station 26075
```

**Меню:** запустите `python simple_tool.py` и выберите пункт **3. Скачать и обработать за один шаг**.

**Из кода Python:**

```python
from weather_tool import fetch_and_process

result = fetch_and_process(2025, 1, 26075)          # один месяц
result = fetch_and_process(2025, [1, 2, 3], 26075)  # несколько месяцев
print(result["processed"], "сроков с инверсиями сохранено")
```

Результат — как и раньше: папка `soundings_ГОД_СТАНЦИЯ/` с отдельным файлом на каждый срок и сводный `DATA.xlsx`. Скачанные исходные данные сохраняются в папке `data/`, поэтому повторная обработка (`--process`) не требует новой загрузки.

> **Запасной вариант (без интернета/если сайт недоступен).** Можно скопировать сроки со страницы сайта в текстовый файл (или Word) и обработать локально: `python weather_tool.py --from-file файл.txt --station 26075`. Главное — захватить строку-заголовок `Observations for Station ... at HH UTC ...` над таблицей.

## Идентификаторы метеостанций

Некоторые часто используемые станции:

- 26075: Санкт-Петербург (Воейково)
- 26702: Москва
- 26850: Казань
- 27612: Екатеринбург
- 31977: Салехард
- 32540: Новосибирск
- 35121: Владивосток

Полный список станций доступен на [сайте Университета Вайоминга](https://weather.arcc.uwyo.edu/upperair/sounding.shtml).

## Структура результатов

После обработки данных вы получите:

1. Отдельные файлы Excel для каждого зондирования (в папке `soundings_YEAR_STATION/`)
2. Общий файл анализа `DATA.xlsx` со следующими листами:
   - `df_full` - полные данные всех инверсий
   - `ground_inv` - только наземные инверсии
   - `elevated_inv` - только приподнятые инверсии
   - `stats` - общая статистика
